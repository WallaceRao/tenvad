注意一下所有任务都需要在cosyvoice的conda环境中完成：conda activate cosyvoice
目标1：
    使用tenvad进行音频文件的VAD检测
    利用fast API实现http服务
    输入：
    audio_file: 音频文件
    输出：
    json格式，分别为人声段开始和结束的时间戳
    其他：
    连续50ms检测到人声则认为开始讲话
    连续50ms检测不到人声则认为开始静音
目标2
    继续完善服务，增加新功能：支持调整音频到指定长度
    调整策略为：
    1） 若目标长度小于当前长度，则需要裁剪音频，假设需要裁剪x秒的音频
      首先检测当前全部音频静音段长度之和，假设总静音长度为y秒
        a) 若y大于x，则按比例计算每段静音裁剪后的长度，假设有一段静音为z秒，则裁剪后静音为z*x/y秒
           根据每段静音长度重建音频文件，取出人声部分，在人声段之间按新的时长增加静音段，注意拼接的地方需要平滑一下。
        b) 若y小于x，表示即使删除全部静音市场，也超过了目标长度，这时则按裁剪90%的比例计算每段静音的新的时长，
           根据每段静音新的长度重建音频文件，取出人声部分，在人声段之间按新的时长增加静音段，注意拼接的地方需要平滑一下。
           之后再利用pytsmod的PV-TSM算法将时长调整为目标长度
    2） 若目标长度于大当前长度，则需要扩充音频，假设需要扩充x秒的音频
      首先检测当前全部音频静音段长度之和，假设总静音长度为y秒
        a) 若y大于x，则按比例计算每段静音扩充后的长度，假设有一段静音为z秒，则扩充后静音为z + z*x/y秒
           根据每段静音长度重建音频文件，取出人声部分，在人声段之间按新的时长增加静音段，注意拼接的地方需要平滑一下。
        b) 若y小于x，则将每段静音时长调整为当前的2倍，
           根据每段静音新的长度重建音频文件，取出人声部分，在人声段之间按新的时长增加静音段，注意拼接的地方需要平滑一下。
           之后再利用pytsmod的PV-TSM算法将时长调整为目标长度
